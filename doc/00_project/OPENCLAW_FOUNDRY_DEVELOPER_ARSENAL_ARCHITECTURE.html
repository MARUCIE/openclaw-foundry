Auth 中间件实现（Hickey: 一个值，不是框架）

// 整个 auth 就这么多
async function authMiddleware(c: Context, next: Next) {
  const token = c.req.header('Authorization')?.replace('Bearer ', '');
  if (!token) return c.json({ error: 'Unauthorized' }, 401);

  const tenant = await c.env.DB.prepare(
    'SELECT * FROM tenants WHERE api_key = ? AND active = 1'
  ).bind(token).first();

  if (!tenant) return c.json({ error: 'Invalid token' }, 401);
  c.set('tenant', tenant);
  await next();
}

商业模式：杠铃策略

安全端（80%）：免费工具 + 内容营销 = 建立信任和流量。激进端（20%）：Arsenal API + 数据授权 = 变现和护城河。不是二选一，是同时存在。

候选模型评估

模型 收入天花板 维护成本 适合独立开发者 PMF 验证难度
A. API 订阅 中 ($5-50/mo/user) 低 YES 中
B. 内容 + 赞助 低-中 ($500-5K/mo) 中（每周写报告） YES 低
C. 平台分成 低 低 YES 高
D. 数据授权 高 ($1-10K/mo/client) 低 YES 高（需数据量）
推荐：杠铃策略（A + B 组合）

安全端（内容 + 免费工具）提供下行保护 -- 即使零付费用户也在建立品牌资产和 SEO 流量。激进端（API 订阅 + 远期数据授权）提供上行收益。两者同时运行，互相增强。
